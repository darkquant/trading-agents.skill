# TradingAgents: 多智能体交易分析技能

一个 [Claude Code](https://claude.ai/code) 技能，编排 10 余个专业 AI 智能体来模拟专业交易公司的运作。灵感来自 [TradingAgents](https://arxiv.org/abs/2412.20138) 研究框架，通过结构化协作、对抗性辩论和逐级审核，生成全面的股票分析报告。

> **免责声明**：这是一个面向教育和研究目的的 AI 工具，不构成任何投资建议。做出投资决策前，请务必咨询合格的金融顾问。

## 工作原理

当你让 Claude Code 分析某只股票时，技能会部署一组子智能体，模拟真实交易台的角色分工：

```
                         用户: "分析 NVDA"
                                |
                   +------------+------------+
                   |            |            |            第一阶段
             基本面分析     技术分析     情绪分析    新闻分析   （并行）
                   |            |            |           |
                   +------------+-----+------+-----------+
                                      |
                             +--------+--------+
                             |                 |         第二阶段
                          多头研究员       空头研究员     （辩论）
                             |                 |
                             +--------+--------+
                                      |
                             研究经理（综合评估）
                                      |
                                   交易员                第三阶段
                                      |
                                  风控经理               第四阶段
                                      |
                               投资组合经理              第五阶段
                                      |
                                 最终建议
```

### 五个阶段

| 阶段            | 智能体                               | 职责                                                                                                  |
| --------------- | ------------------------------------ | ----------------------------------------------------------------------------------------------------- |
| **1. 分析**     | 基本面、技术面、情绪面、新闻面分析师 | 四位分析师**并行**运行，基本面分析师通过 MCP 数据源获取机构级金融数据，技术面分析师通过 yfinance 获取行情数据，其余通过网络搜索采集信息。 |
| **2. 研究辩论** | 多头研究员、空头研究员、研究经理     | 多头和空头研究员进行可配置轮次的辩论（默认 1 轮），研究经理随后将辩论综合为平衡性摘要。               |
| **3. 交易决策** | 交易员                               | 综合所有报告，给出 BUY（买入）/ SELL（卖出）/ HOLD（持有）建议，包含入场/出场点位及仓位规模。         |
| **4. 风险审查** | 风控经理                             | 评估仓位风险、波动性、事件风险、集中度风险及下行场景。                                                |
| **5. 组合审批** | 投资组合经理                         | 做最终决定：APPROVE（批准）/ MODIFY（修改）/ REJECT（驳回）。                                         |

## 安装

### 前置条件

- [Claude Code](https://claude.ai/code)（CLI、桌面应用或 IDE 插件）
- Python 3.9+（用于辅助脚本）

### 安装技能

在你的项目 `.claude/settings.json` 中添加技能路径：

```json
{
  "skills": ["/path/to/trading-agents.skill"]
}
```

也可添加到用户级配置 `~/.claude/settings.json`，使其全局可用。

使用辅助脚本前，请先运行 `uv sync` 以安装所需的 Python 依赖（如 `yfinance`、`numpy`、`pandas`）。

## 使用方法

直接向 Claude Code 提出股票分析请求即可，技能会自动触发。

### 示例提示

**基础分析：**

```
帮我分析一下英伟达（NVDA），现在是买入的好时机吗？
```

**指定辩论轮次：**

```
对特斯拉（TSLA）做一次完整分析，用 3 轮多空辩论。
```

**带投资者背景：**

```
分析一下苹果（AAPL）。我是保守型投资者，看长期。
目前已持有较多科技股，担心集中度风险。
```

**English prompt also works:**

```
Run a full analysis on NVDA with 2 rounds of debate.
```

### 输出内容

1. **各分析师报告**（保存到工作目录）：
   - `fundamental_analysis.md` — 估值、财务健康、盈利质量
   - `technical_analysis.md` — 价格走势、技术指标（RSI、MACD、布林带）、支撑/阻力位
   - `sentiment_analysis.md` — 社交媒体情绪、分析师评级、机构动向
   - `news_analysis.md` — 近期动态、宏观背景、即将到来的催化事件

2. **辩论记录**（`debate_record.md`）— 完整的多空对抗辩论过程

3. **研究摘要**（`research_summary.md`）— 研究经理的平衡性综合评估

4. **交易建议**（`trading_recommendation.md`）— BUY/SELL/HOLD 及入场/出场策略

5. **风险评估**（`risk_assessment.md`）— 风险等级及下行场景分析

6. **组合决策**（`portfolio_decision.md`）— 投资组合经理的最终裁决

7. **对话摘要** — 在聊天中直接展示最终建议、核心理由、关键风险和置信度

## 智能体列表

| 智能体           | 角色                                             | 数据来源                                                  |
| ---------------- | ------------------------------------------------ | --------------------------------------------------------- |
| **基本面分析师** | 评估财务健康、估值、盈利质量、竞争地位           | MCP 数据源（S&P Global、FactSet、Daloopa、Morningstar、LSEG）、SEC 文件、公司投资者关系页面 |
| **技术面分析师** | 分析价格形态、动量指标、成交量、波动性           | yfinance、技术指标脚本、图表形态分析                      |
| **情绪面分析师** | 从社交媒体、分析师评级、机构资金流向判断市场情绪 | Reddit、Twitter/X、StockTwits、雪球、分析师报告、13F 文件 |
| **新闻面分析师** | 评估近期新闻、宏观事件、行业动态、催化事件       | Reuters、Bloomberg、FT、WSJ、财新、第一财经、公司公告     |
| **多头研究员**   | 构建最强有力的看涨论据                           | 全部分析师报告 + 前轮辩论记录                             |
| **空头研究员**   | 构建最强有力的看跌论据                           | 全部分析师报告 + 前轮辩论记录                             |
| **研究经理**     | 客观综合多空辩论                                 | 辩论记录 + 分析师报告                                     |
| **交易员**       | 给出可执行的 BUY/SELL/HOLD 建议及仓位            | 全部报告 + 研究摘要                                       |
| **风控经理**     | 评估风险：仓位、波动、事件、尾部风险             | 交易建议 + 全部报告                                       |
| **投资组合经理** | 最终决策者：APPROVE / MODIFY / REJECT            | 以上所有内容                                              |

## 辅助脚本

两个 Python 脚本为分析师智能体提供数据支持：

```bash
# 获取价格历史、财务报表和关键指标
python scripts/fetch_market_data.py TICKER [-o OUTPUT_DIR]

# 计算 RSI、MACD、布林带、移动平均线等技术指标
python scripts/technical_indicators.py TICKER [-o OUTPUT_DIR]
```

两个脚本都会在缺少依赖时自动安装 `yfinance`（技术指标脚本还会安装 `numpy`），输出为 JSON 格式。

## 数据质量标准

所有报告遵循严格的引用规范：

- **优先使用一手来源**：公司公告、交易所文件、官方财报
- **每个关键数据点必须注明来源**，附带可点击的 URL、报告期间和货币/单位
- **新闻引用权威媒体**：Reuters、Bloomberg、FT、WSJ、财新、第一财经（优先于博客/社交媒体）
- **情绪判断必须归因到具体来源**，不得使用模糊表述
- **技术指标配有通俗解释**，帮助非专业读者理解每个指标的含义

## 成本提示

每次完整分析会产生 **8-12+ 个子智能体**，每个智能体会执行多次工具调用（网络搜索、脚本执行、文件写入）。分析多只股票时，建议逐一进行。
